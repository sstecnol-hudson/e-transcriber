🎯 OBJETIVO
-----------
Verificar se o carregamento de protocolos foi corrigido e o fluxo de 
qualificação funciona corretamente.

⚡ TESTE RÁPIDO (2 MINUTOS)
----------------------------

1. Abra um terminal/prompt de comando

2. Navegue até a pasta do projeto:
   cd "c:\Projects\e-transciber +"

3. Inicie um servidor local (escolha uma opção):

   OPÇÃO A - Python (se tiver instalado):
   python -m http.server 8000
   
   OPÇÃO B - Node.js (se tiver instalado):
   npx http-server
   
   OPÇÃO C - VS Code Live Server:
   - Clique com botão direito em index.html
   - Selecione "Open with Live Server"

4. Abra o navegador em:
   http://localhost:8000/TEST_PROTOCOLS.html

5. Verifique os resultados:
   ✅ "Sistema inicializado com sucesso!" - OK
   ✅ "Todos os protocolos carregados!" - OK
   ✅ "Fluxo iniciado com sucesso" - OK

🧪 TESTE MANUAL (5 MINUTOS)
----------------------------

1. Abra o navegador em:
   http://localhost:8000/index.html

2. Preencha os dados de uma consulta:
   - Nome do paciente: "Teste"
   - Queixa principal: "Dor de cabeça"
   - Descrição: "Teste de qualificação"

3. Clique em "Qualificar para Encaminhamento"
   ✅ Deve aparecer um modal com 3 especialidades

4. Clique em uma especialidade (ex: "Endocrinologia")
   ✅ Deve aparecer um questionário

5. Se tudo funcionou, o problema foi corrigido! 🎉

🔍 VERIFICAR CONSOLE (AVANÇADO)
--------------------------------

1. Abra http://localhost:8000/index.html

2. Pressione F12 para abrir DevTools

3. Vá para a aba "Console"

4. Procure por estas mensagens (devem aparecer):
   ✅ "✅ Protocolos carregados do localStorage"
      OU
   ✅ "✅ Protocolo carregado: endocrinologia"
   ✅ "✅ Protocolo carregado: cardiologia"
   ✅ "✅ Protocolo carregado: reumatologia"
   ✅ "✅ Sistema de qualificação pronto para uso"

5. Se vir estas mensagens, os protocolos foram carregados com sucesso!

❌ SE NÃO FUNCIONAR
-------------------

1. Verifique se está usando um servidor local (não file://)
   - Abra DevTools (F12)
   - Vá para "Console"
   - Se vir erro CORS, está usando file://
   - Solução: Use um servidor local (veja "TESTE RÁPIDO" acima)

2. Limpe o cache:
   - Pressione Ctrl+Shift+Delete
   - Selecione "Limpar tudo"
   - Recarregue a página

3. Limpe o localStorage:
   - Abra DevTools (F12)
   - Vá para "Application"
   - Clique em "Local Storage"
   - Clique com botão direito e "Clear All"
   - Recarregue a página

4. Verifique se os arquivos JSON existem:
   - Abra: modules/qualification/data/
   - Deve ter 3 arquivos:
     ✅ protocol-endocrinologia.json
     ✅ protocol-cardiologia.json
     ✅ protocol-reumatologia.json

5. Se ainda não funcionar:
   - Abra DevTools (F12)
   - Vá para "Console"
   - Copie as mensagens de erro
   - Procure por "❌" ou "Error"

📊 CHECKLIST DE SUCESSO
-----------------------

Após as correções, você deve ver:

✅ Página carrega sem erros
✅ Console mostra "✅ Protocolo carregado: ..."
✅ Botão "Qualificar para Encaminhamento" funciona
✅ Modal de especialidades aparece
✅ Questionário aparece após selecionar especialidade
✅ Sem mensagens de erro no console
